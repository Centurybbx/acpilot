# 08. 测试策略与迭代治理

## 1) 测试策略（补齐原计划缺失）

### Daemon 层
- ACP 协议兼容测试：mock agent 驱动 `initialize/new/prompt/cancel`
- 进程管理测试：崩溃重启、超限熔断、空闲回收
- 鉴权与安全测试：首次配对、已配对自动登录、设备撤销、无权限访问

### Web UI 层
- 动态渲染测试：configOptions/modes/commands 的可见性规则
- 权限审批流测试：请求展示、审批回传、错误处理
- 移动端交互测试：键盘遮挡、窄屏折叠区、重连提示状态

### 端到端
- Codex 完整闭环 E2E
- Claude/Copilot 基础会话 E2E
- 断连恢复 E2E（网络抖动脚本）

## 2) 发布门禁
- 单元 + 集成 + E2E 全绿
- 关键路径性能达标（首屏、流式延迟、重连耗时）
- 安全检查通过（鉴权、日志脱敏、限流）
- daemon 重启后 trusted device 关系保持不变

## 3) 迭代治理规则
- 每个迭代必须产出：
  - 阶段状态更新（`04_phase_plan_v2.md`）
  - 能力矩阵更新（`05_agent_integration_matrix.md`）
  - 风险台账更新（本文档）
- 未通过门禁不得升级阶段状态为完成。

## 4) 风险台账（初始）
| 风险 | 影响 | 应对策略 |
|---|---|---|
| Copilot 命令发现不稳定 | slash 功能不可用 | MVP 排除 + 动态降级 |
| ACP 协议字段变更 | 兼容性中断 | 协议版本冻结 + 兼容层 |
| 手机端频繁断连 | 对话中断 | 重连 + 事件补齐 + 恢复策略 |
| 日志泄露敏感信息 | 安全事故 | 强制脱敏 + 审计分级 |
| 认证状态随 daemon 重启丢失 | 手机端需要反复手工登录 | 持久化服务端密钥与 trusted device store |
